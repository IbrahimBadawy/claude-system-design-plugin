9 Auto-Invoked Skills

Skills activate automatically based on context. No commands needed.

design-system

Triggers when designing new systems. Uses 4-step framework, references 16 design patterns, searches for open source alternatives.

evaluate-system

Triggers when reviewing systems. Multi-dimensional evaluation using criteria from all 8 books. Score card with prioritized findings.

implement-system

Triggers when coding. Writes real production code. Researches docs before using any library. Follows Clean Architecture.

research-tech

Triggers when comparing technologies. Searches the web, reads official docs, synthesizes findings into comparison matrices.

project-manager

Triggers for project management. Tracks multiple projects, manages plans, templates, and dependencies between projects.

documentation

Triggers when generating docs. Recommends Swagger, Storybook, TypeDoc, Docusaurus based on stack. Auto-detects undocumented code.

frontend-design

Triggers for UI work. Asks about style preferences, picks best component library (shadcn, PrimeVue, MUI, etc.), enforces accessibility.

devops

Triggers for infrastructure. Generates Docker, CI/CD, deployment configs. Supports GitHub Actions, AWS, Kubernetes, Terraform.

opensource-research

Triggers when looking for existing solutions. Compares open source systems, analyzes licenses (MIT/GPL/AGPL), recommends build vs buy.

16 Always-Active Rules

These rules are enforced automatically on every interaction. Non-negotiable quality standards.

01Requirements First — never code without understanding the problem
02Design Before Code — API + data model + architecture first
03Justify Decisions — every choice needs a "why" with trade-offs
04Think About Failure — timeouts, retries, circuit breakers, degradation
05Security by Default — auth, encryption, validation in every layer
06Clean Code — naming, functions, error handling, DRY, SOLID
07Scalability — design for 10x, build for 1x, don't over-engineer
08Observability — metrics (RED), logging (JSON), tracing, alerting
09Data Model is King — access patterns first, schema second
10API Standards — versioning, pagination, idempotency, validation
11Research Before Code — fetch latest docs before using any library
12Auto Documentation — docs generated with code, auto-update triggers
13No Code Without Plan — all plans approved + explicit user command
14Domain Modeling — DDD: bounded contexts, aggregates, events
15Accessibility — WCAG 2.2 AA, keyboard nav, screen readers, RTL
16Performance Budgets — API p99, Core Web Vitals, bundle size limits

Get Started

Set up the plugin in minutes.

Step 1: Clone or copy the plugin

# Copy the entire directory to your project
cp -r system-design-plugin/.claude /your-project/
cp system-design-plugin/CLAUDE.md /your-project/

Step 2: Install companion tools

# Essential companions
/plugin install context7
npm i -g uipro-cli && uipro init --ai claude
/plugin install security-guidance
/plugin install code-review
/plugin install playwright

Step 3: Open in Claude Code

# Open your project directory
claude /your-project

# Start designing!
/project add my-system
/design my-system
