# /i18n - Internationalization & RTL Setup

Set up, audit, or extend internationalization. Enforces CSS logical properties,
ICU MessageFormat, `Intl.*` APIs, icon mirroring rules, and RTL testing.

## Usage
```
/i18n setup                          # Scaffold i18n + RTL from scratch
/i18n add <locale>                   # Add a new locale (e.g. /i18n add ar)
/i18n audit                          # Scan for hardcoded strings, physical CSS, concat translations
/i18n rtl-check                      # Verify every page in RTL mode
/i18n extract                        # Pull keys from source to translation files
/i18n missing                        # Show untranslated keys per locale
/i18n font-stack <locale>            # Recommend font stack for a locale
```

## What `/i18n setup` creates

Based on the chosen framework:

### Next.js App Router (with `next-intl`)
```
src/
  i18n/
    config.ts                # locales: ['en', 'ar', 'fr']
    request.ts               # server-side locale loader
    navigation.ts            # locale-aware Link, redirect
  messages/
    en.json
    ar.json
    fr.json
app/
  [locale]/
    layout.tsx               # sets <html lang dir>
    page.tsx
middleware.ts                # locale detection
```

### Vue / Nuxt (with `vue-i18n`)
```
locales/
  en.json
  ar.json
plugins/i18n.ts
composables/useI18n.ts
```

### Any framework
- Font stack CSS per locale via `:lang()` / `[lang]` selectors
- Global CSS guard for `prefers-reduced-motion`
- Utility components: `<FormatNumber>`, `<FormatDate>`, `<FormatCurrency>`, `<PluralCount>`
- ESLint rules:
  - `no-hardcoded-strings`
  - `no-physical-margin` (require logical properties)
  - `no-string-concat-in-i18n`

## `/i18n rtl-check`

Forces RTL mode and takes Playwright snapshots of every page. Reports:

```
RTL Audit:
- 28/34 pages render correctly in RTL
- 6 pages have issues:
  1. /dashboard — KPI cards overlap at 320px RTL
  2. /settings/security — icon flipped incorrectly (clock shouldn't flip)
  3. /checkout — number formatting shows Western digits in ar locale (use Intl)
  ...
```

## Font Stacks

`/i18n font-stack ar` output:
```css
/* Arabic font stack — recommended */
[lang="ar"] {
  font-family:
    "IBM Plex Sans Arabic",
    "Noto Sans Arabic",
    "Tajawal",
    "Cairo",
    "Segoe UI",
    Tahoma,
    sans-serif;
  line-height: 1.75;  /* Arabic needs more */
  font-feature-settings: "kern", "liga";
}

[lang="ar"] strong, [lang="ar"] b {
  font-weight: 500;  /* 500 looks bold enough */
}
```

## Translation Workflow

1. Developer writes `t('auth.login.submit')` in source
2. CI runs `/i18n extract` to pull all keys
3. Keys uploaded to Lokalise / Crowdin / Phrase with context + screenshots
4. Translators translate
5. CI pulls translations back, updates `messages/*.json`
6. CI fails if keys are missing in any locale

## Integration with Other Commands

- `/pages` — reminds to localize page titles + meta
- `/implement` — uses `t()` helpers for all user-facing strings
- `/frontend design` — checks font stack is locale-appropriate
- `/a11y-audit` — ensures `<html lang dir>` is set
- `/copy-audit` — checks for idioms that don't translate
